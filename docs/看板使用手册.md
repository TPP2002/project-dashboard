# 项目管理看板 · 使用手册（大白话版）

这份手册教你怎么**打开看板、看懂每一屏、直接在网页上拍板**。全程不用懂代码，跟着点就行。

> 一句话介绍：这是一个网页，把你几个项目里"每件事做到哪一步了、卡在哪、有什么等你拿主意"一屏看清楚。

## 一、怎么打开看板

**最省事的办法——双击一个文件就行：**

| 你的电脑 | 双击这个文件 |
|---|---|
| Windows | 看板根目录里的 **`启动看板.bat`** |
| Mac / Linux | 看板根目录里的 **`dashboard.sh`**（或在终端里运行 `./dashboard.sh`） |

> 「看板根目录」就是装看板程序的那个文件夹，路径一般是 `C:\Users\你的用户名\.claude\dashboard\`。

**双击之后会自动发生这几件事**（你不用管，看着就行）：

1. **第一次用**会先装一次"界面零件"（只装一次，可能要等一两分钟，屏幕上有进度）；以后再打开就没这一步了。
2. 生成最新的网页界面。
3. 启动看板服务，然后**自动帮你打开浏览器**。

浏览器打开后，地址通常是 **`http://127.0.0.1:6060/`**（以那个黑色小窗口里显示的地址为准——如果 6060 被别的程序占了，它会自动换成 6061、6062……）。

**关掉看板**：把那个黑色小窗口关掉，或在里面按 `Ctrl + C`。关掉后网页就打不开了，想再看重新双击一次即可。

**已经开着、又双击了一次**：不会重复开两个，它会发现"已经在跑了"，直接再帮你把浏览器打开到同一个页面。所以放心多点几下也不会乱。

## 二、界面整体长什么样

打开后，屏幕分三块：

```
┌─────────────────────────────────────────────┐
│ 顶栏： 📋项目看板   [项目下拉▾]   ●实时  ↻   ❓3 │   ← 最上面一条
├──────────┬──────────────────────────────────┤
│ 左侧导航  │                                  │
│ 🏠 总览   │                                  │
│ 📋 看板   │          主区（正文）             │
│ ❓ 待拍板 │      （点左边不同的菜单，          │
│  视图     │        这里换成对应的内容）        │
│ 📜 活动流 │                                  │
│ ⚠️ 风险   │                                  │
│  …       │                                  │
└──────────┴──────────────────────────────────┘
```

- **顶栏**（最上面一条）：
  - 左边 **项目下拉框**：切换你现在在看哪个项目。
  - **● 小圆点 + 文字**：网页和后台的连接状态（绿=实时，见[第十节](#十页面会自己刷新实时)）。
  - **↻ 刷新按钮**：手动刷新一下（一般不用点，它会自己刷）。
  - **❓ 铃铛**：待拍板提醒。右上角有个**橙色数字**就代表"有几件事等你拿主意"，点它直接去待拍板中心。
- **左侧导航**：一列菜单，点哪个，右边主区就换成哪个内容。最上面三个（总览 / 看板 / 待拍板）是最常用的，下面「视图」那一组是进阶看板。
- **主区**：正文，占最大那块。

## 三、总览页：一眼看全部项目

左侧点 **🏠 总览**（打开看板默认就是这一页）。

**最上面一排大数字**，是把你所有项目加在一起的总账：

- **项目**：一共几个项目。
- **任务**：一共多少件事。
- **已完工**：做完了多少件（绿色）。
- **待拍板**：还有多少件等你拿主意（有数字时会变**橙色**提醒你）。

**下面是一张张项目卡片**，每个项目一张：

- 左边一个**圆环**：这个项目的完成度百分比，圈里写着「做完的件数 / 总件数」。
- 右边：**项目名**、它在电脑上的**文件夹路径**；如果有待拿主意的事，会有一行橙色的「❓ 几条待拍板」。
- 中间一排**彩色小标签**：这个项目各个状态各有几件（比如「🔨 施工中 3」「✅ 已完工 12」），颜色含义见[第七节](#七状态的颜色和图标是什么意思)。
- 最下面**最近动态**：这个项目最近发生的两三件事。

**点任意一张项目卡片**，就会切到该项目、并跳到它的[看板页](#四看板页看单个项目的进度墙)。

## 四、看板页：看单个项目的进度墙

左侧点 **📋 看板**。这一页把当前项目的所有事，**按状态分成一列一列**（每一列叫一条"泳道"），像贴便利贴的白板：

```
⬜ 未开工   🔨 施工中   🏁 收官     ✅ 已完工
┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐
│ 卡片  │  │ 卡片  │  │ 卡片  │  │ 卡片  │
│ 卡片  │  │ 卡片  │  │       │  │ 卡片  │
└───────┘  └───────┘  └───────┘  └───────┘
```

- 每一列顶上写着**这个状态的图标、名字、有几件**。只有**有内容的列才会显示**（空的状态不占地方）。列多的时候可以**左右滚动**。
- 每张**卡片**上：
  - **任务编号**（比如 `P17`）和**标题**。
  - 一条**进度条**（做了多少）。
  - 底部一些**小标签**：分支 🌿、`PR #编号`、`W波次号` 等（没有就不显示）。
  - 右上角如果有 **❓+数字**，表示这件事上有几个问题等你拍板。
  - 卡片**左边那条竖色条**＝它的状态颜色。
- **点一张卡片**，右边会滑出「详情抽屉」，看这件事的全部信息（见下一节）。

顶上还有一行「X/Y 完工 · Z%」，是这个项目的总进度。

## 五、卡片抽屉：看一件事的全部细节

在总览、看板或搜索页里**点一张卡片**，屏幕**右侧会滑出一个抽屉**，从上到下可能有这些内容（有才显示）：

- **标题、进度条、一段说明**：这件事是什么、做到哪了。
- **基本信息**：波次、设计/开工/完工日期、类型检查是否通过、下一个小目标。
- **测试**：一共多少条、通过多少条。
- **分支 / 占用**：用了哪些代码分支 🌿、工作区 🌲、PR、涉及哪些文件 📁、哪些地方是"禁区不能碰" ⛔。
- **依赖 / 阻塞**：这件事要等哪件先做完、被谁卡住了、为什么暂缓。
- **决策**：需要拿主意的问题。**还没定的可以当场拍板**（做法和[待拍板中心](#六待拍板中心怎么在网页上直接拿主意)一样）；已经定了的会显示「✅ 已拍板：你选的答案」。
- **文档**：相关的说明文件。点文件名可以**就地展开预览**，也可以点「打开 ↗」在新标签页看。
- **活动**：这件事的时间线流水账——谁、什么时候、做了什么。

**关掉抽屉**有三种办法：点右上角 **✕**、点抽屉**左边的灰色空白处**、或按键盘 **`Esc`** 键。

## 六、待拍板中心：怎么在网页上直接拿主意

**"拍板"就是替某个卡住的问题做决定。** 施工的人遇到"有几种做法、得您定"的岔路口，就会在这里挂一个问题等你选。

**怎么进去**：点顶栏右上角的 **❓ 铃铛**（有橙色数字代表有几条），或点左侧导航的 **❓ 待拍板**。

**这一页把所有项目里"等你拿主意"的问题都列出来**，每一条是一张卡：

- 上面一行：**哪个项目、哪件事（编号+标题）、问题编号**。
- 中间：**问题本身**（要你定什么）。
- 下面：**几个选项**，每个选项是一个可以点的按钮；其中系统**推荐**的那个会有一个绿色的「推荐」小标，并且**默认already帮你选好了**。

**拍板三步走**：

1. （可选）在右上角 **「拍板人」** 填上你的名字/署名——默认写着「看板」，想留个名就改一下。
2. **点一个选项**把它选中（选中的前面是实心圆点 **●**，没选的是空心 **○**）。想用推荐项就不用点，它已经默认选好了。
3. 点右下角蓝色的 **「确认拍板：你选的答案」** 按钮。

**拍完会怎样**：几秒钟内整个界面自动刷新，这条问题**标上你选的答案、从待拍板清单里消失**，铃铛上的数字减一，看板也会自动记一笔"谁在什么时候拍了什么"。

> 小提示：任务卡片**整体的状态**（比如从"待拍板"变成别的）**不会因为你拍板就自动改变**——那一步由后续施工的人来推进。你这一步只是"把答案定下来"，让活儿能往下走。

**也可以在卡片抽屉里就地拍板**：点开任意一张卡片，抽屉里「决策」那一段，没定的问题同样能选项+确认，效果一样。

## 七、状态的颜色和图标是什么意思

看板、总览、卡片上到处用**颜色+图标**表示一件事处在哪个阶段。对照这张表：

| 图标 | 状态 | 颜色 | 大白话意思 |
|:---:|---|---|---|
| ⬜ | 未开工 | 灰 | 还没排上，连要不要做、怎么做都没定 |
| 📋 | 待开工 | 蓝 | 已经排好队，等着开工 |
| ❓ | 待拍板 | 橙（提醒色） | 卡在一个"要你拿主意"的问题上 |
| ✅ | 已拍板 | 绿 | 你已经拍了板，可以往下走了 |
| 🔨 | 施工中 | 琥珀黄 | 正在做 |
| 🔄 | 可复工 | 青 | 之前停下的，现在可以接着做 |
| 🏁 | 收官 | 绿 | 快做完了，在收尾 |
| ✅ | 已完工 | 绿 | 全部做完 |
| 🚫 | 暂缓 | 红 | 先搁一搁，暂时不做 |
| 🎬 | 压轴 | 紫 | 留到最后压轴做的大活 |

> 「已拍板」和「已完工」都用 ✅，但名字和位置不一样，别弄混：一个是"决定定了"，一个是"活干完了"。
> 记不住也没关系——**橙色 = 要你出手**、**绿色 = 好了/放心**、**红色 = 停住了**，抓这三个大方向就够用。

## 八、其它几个视图，一句话看懂

左侧导航「视图」那一组，都是换个角度看同一批事，随便点着看：

| 菜单 | 看什么 |
|---|---|
| 📜 活动流 | 一条时间线，最近谁做了什么，像流水账 |
| ⚠️ 风险 | 被搁置、被卡住、暂缓的事集中在这儿，一眼看哪些"堵着" |
| 🌊 波次 | 按"第几批做"分组看，哪些是先干的、哪些排后面 |
| 🚦 验收 | 红黄绿灯表格，看每件事的质量检查过没过 |
| 💥 占用 | 看有没有两件事"抢同一个东西"（同一分支/文件），防止打架 |
| 📅 甘特 | 时间条形图，看事情在时间上的先后和长短 |
| 🕸️ 依赖 | 关系图，看哪件事要等哪件先完成 |
| 🔍 搜索 | 按名字/编号找事、按状态等条件筛，还能导出快照 |

> 甘特图和依赖关系图带图表，**点进去时才现加载**，第一次可能慢半秒，正常。

## 九、多个项目怎么来回切

- **看某一个项目**：点**顶栏左边的项目下拉框**，选哪个就切到哪个，看板/待拍板等页面都会跟着换成那个项目。
- **一次看全部项目**：回到 **🏠 总览**，所有项目的卡片一起摆出来，点任意一张就进去。

新项目要先由技术那边"登记"一次才会出现在这里（一次性设置，你自己不用管）。如果总览页空空的写着"暂无项目"，就是还没登记，找一下帮你搭看板的人登记即可。

## 十、页面会自己刷新（实时）

看板**不用你手动刷新**——施工那边一有进展，页面几秒内自己更新，变化的卡片还会**黄色闪一下**提醒你"这张刚变了"。

顶栏那个**小圆点**告诉你连接状态：

| 圆点 | 文字 | 意思 |
|:---:|---|---|
| 🟢 绿 | 实时 | 连得好好的，一有变化立刻推给你（最理想） |
| 🟠 橙 | 轮询 | 退而求其次，每隔几秒自己去问一次有没有更新（也够用） |
| 🔵 蓝（闪） | 连接中 | 正在连，等一下 |
| ⚪ 灰 | 离线 | 没连上，可能是后台关了——重新双击启动一下 |

实在想立刻刷，点顶栏的 **↻** 按钮也行。

## 十一、常见问题

**Q：双击 `启动看板.bat`，黑框闪一下就没了 / 报错。**
多半是**没装 Node.js**（看板运行要用到它）。黑框里如果写着找不到 `node`，去 <https://nodejs.org/> 装一个（选"LTS"长期支持版），装完再双击。若写着安装依赖失败，一般是**网不好**，连上网再试。

**Q：服务起来了，但浏览器没自动打开。**
自己打开浏览器，在地址栏输入黑框里显示的地址（通常是 `http://127.0.0.1:6060/`）回车即可。

**Q：网页打开是一句"服务已就绪，但界面还没构建"。**
说明"网页界面"还没生成好。正常走双击启动器会自动生成；如果你是手动起的服务，就先在 **`web` 文件夹**里跑一次 `npm run build`（第一次还要先 `npm install`），再刷新网页。或者干脆用启动器（`启动看板.bat` / `dashboard.sh`），它会替你把这步做掉。

**Q：黑框里说"端口 6060 被占用"。**
不用管，它会自动换下一个端口（6061、6062…），**以黑框里最终显示的那个地址为准**去访问就行。

**Q：数字/状态看着不对、像是旧的。**
先看顶栏圆点是不是"离线/轮询"；点一下 **↻** 手动刷新；还不对就把黑框关掉、重新双击启动一次。

**Q：待拍板拍错了，能改吗？**
网页上拍板是"一锤定音"、当场落库，界面上不能反悔。真拍错了，找帮你搭看板的技术那边用命令改一下即可（这属于少见情况）。

**Q：手机上能看吗？**
默认只能**本机（这台电脑）**打开，地址里的 `127.0.0.1` 就是"只限本机"的意思，手机/别的电脑访问不到。这是为安全考虑，正常用途够了。

**Q：怎么彻底关掉看板？**
关掉那个黑色小窗口，或在里面按 `Ctrl + C`。关掉后网页就失效了，想再用重新双击启动器即可。看板不会偷偷在后台常驻。
